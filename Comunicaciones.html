Comunicaciones

Introducción:

En el contexto de la Industria 4.0, las comunicaciones industriales son fundamentales para la interconexión de dispositivos, sistemas y procesos dentro de entornos industriales. Este campo se enfoca en facilitar la transferencia de datos y la comunicación entre maquinaria, equipos, sensores, sistemas de control y otros dispositivos en entornos de fabricación y producción.

Algunos aspectos clave de las comunicaciones industriales en la Industria 4.0 incluyen:

  • Interoperabilidad: El objetivo es lograr la interoperabilidad entre equipos y sistemas heterogéneos, permitiendo que dispositivos de diferentes fabricantes y tecnologías se comuniquen entre sí de manera eficiente y sin problemas.

  • Protocolos de comunicación: Se utilizan diversos protocolos de comunicación como OPC UA, Modbus, PROFINET, EtherNet/IP, MQTT, entre otros, para facilitar la transferencia de datos de forma segura y eficiente en entornos industriales.

  • Internet de las cosas (IoT) e Internet Industrial de las Cosas (IIoT): Estas tecnologías permiten la conectividad y comunicación entre dispositivos, sensores y sistemas, permitiendo la recolección de datos en tiempo real para análisis y control.

  • Computación en la nube y edge computing: Facilitan el almacenamiento, procesamiento y análisis de grandes volúmenes de datos generados por dispositivos industriales, lo que permite tomar decisiones informadas de manera más ágil y eficiente.

  • Seguridad de las comunicaciones: Dado que la conectividad expone los sistemas industriales a riesgos de seguridad, se hace hincapié en implementar medidas de seguridad robustas para proteger los datos y los sistemas contra amenazas cibernéticas.

  • Comunicación hombre-máquina (HMI): Las interfaces hombre-máquina juegan un papel crucial al permitir a los operarios y personal de planta interactuar con los sistemas de control y monitoreo para supervisar y controlar los procesos de fabricación.

En el caso particular del proyecto se deben crear comuniaciones para los siguientes entornos de desarrollo:

Ignitition:

Ignition es una plataforma de software para la automatización industrial y la gestión de la información que ofrece una solución integral para supervisar, controlar y analizar sistemas en entornos industriales. Esta plataforma permite la integración de dispositivos y sistemas heterogéneos, ofreciendo herramientas de desarrollo y despliegue rápido de aplicaciones, así como capacidades avanzadas de visualización de datos en tiempo real, análisis, gestión de alarmas y reportes. Ignition se destaca por su arquitectura modular, flexibilidad, escalabilidad y su capacidad para conectar y gestionar equipos y procesos en una amplia gama de sectores industriales.

Ignition para nuestro caso, sera el centro de todas nuestras comunicaciones entre los diferentes entornos de desarrollo, esta aplicación al ser diseñada con este fin cuenta con bastante variedad de protocolos de comuniación y habilita suficientes apliaciones de interes en el área de la Mecatronica.

Studio5000:

Studio 5000 es un entorno de desarrollo de software de Rockwell Automation diseñado para la programación, configuración y mantenimiento de sistemas de control industrial, especialmente para los controladores de automatización programables (PLCs) y sistemas de control distribuido (DCS). Esta plataforma integra herramientas de diseño, simulación, programación, diagnóstico y optimización de sistemas de automatización, permitiendo a los ingenieros y profesionales de control crear y mantener aplicaciones de control complejas, ofreciendo una interfaz unificada para la configuración y programación de dispositivos, así como la gestión de redes y sistemas de control en entornos industriales.

Studio5000 tiene un modulo dedicado a la conectividad y comunicaciones en diferentes entornos de desarrollo el cual es RSLinx,un software de comunicaciones de Rockwell Automation utilizado en entornos industriales para establecer la conectividad y la comunicación entre diferentes dispositivos de control, como controladores programables (PLCs) y computadoras, a través de diferentes protocolos de comunicación. Este software proporciona una interfaz para la configuración y administración de conexiones de red, permitiendo la recolección de datos en tiempo real, el monitoreo de dispositivos y la transferencia de información entre equipos, lo que facilita el control y la supervisión de sistemas industriales en entornos de fabricación y automatización.

Siemens NX:

Siemens NX es un software integral de diseño, fabricación y simulación utilizado en industrias como la automotriz, aeroespacial y de maquinaria. Ofrece herramientas avanzadas para modelado 3D, análisis, simulación de rendimiento y fabricación asistida por computadora. Destacándose por su capacidad de integrar diferentes etapas del proceso de diseño y fabricación en una única plataforma, fomentando la colaboración entre equipos y proporcionando una suite completa para el desarrollo de productos, desde la conceptualización hasta la fabricación final.

Para lograr comunicación con nx, se debe conectar Studio5000 el cual coordinara toda la logica y configuración de la simulación previamente creada.

RobotStudio:

RobotStudio es la aplicacion por la cual se programan y se visualizan las rutinas de movimiento de los robots industruales de la familia ABB.

para la comunicación con RobotStudio, contamos con el protocolo de comunicaciones industriales OPC-UA, el cual permite conección directa entre ignition y robotStudio

Implementación en proyecto:

En primera instancia se intento multiples ocaciones y de diferentes formas establecer conexión con los diferentes entornos de desarrollo,También,se consulto a los profesores sobre las dificultades que presentamos.

Lastimosamente no logramos entablar una conexión entre los entornos de desarrollo y la conclusión a la cual llegamos despues de varios intentos fallidos y dialogarlo con los profesores. Es que se debe a incompatibilidad de los programas o mala instalación de los mismos (pese a que se reinstalo en reiteradas ocasiones)
